# Design Context

## Users

2–10 known people per deployment (self-hosted). They use the app on phones — often as a home-screen PWA — to: request approval for a purchase ("is this OK?"), approve/deny someone else's request in one tap, log what they spent, watch a budget, and occasionally hide a gift purchase from a partner ("seal"). Context is quick, glanceable, thumb-driven, frequently one-handed. The single most-used interaction is answering an approval request; the second is scanning "where did the money go."

## Brand Personality

Calm, precise, trustworthy, native. Three words: **quiet, physical, exact.** Money and approval state must feel authoritative and never jittery. The interface should feel like a well-made object, not a web page — closer to a Wallet pass than a dashboard. It should never nag, never shout; it earns trust by being legible and honest (see sealing: "a surprise, not a hidden ledger").

## Aesthetic Direction

**"Paper Statement" — a warm printed-paper ledger crossed with a magazine.** History: a faithful dark iOS/Wallet clone was tried and rejected as too generic; a bolder dark "Onyx" pass was _still_ rejected as generic; the user then chose a **light "paper statement"** direction, which landed. Theme is **light**, always (a deliberate, distinctive choice — not a safe default).

- **Type is the identity.** Body/UI = **Hanken Grotesk**; masthead headings + all hero/ring **numbers** = **Bodoni Moda** (a high-contrast didone almost no app uses → reads as designed, not templated). Both self-hosted via `@fontsource-variable`. Big money is set in the serif as a magazine headline; small list amounts stay grotesque-tabular for legibility. `.num` sets tabular only; **font is inherited from context** (add `font-[family-name:var(--font-display)]` to make a number serif).
- **Warm paper palette** (OKLCH, hue ~85): `--paper` cream canvas, `--ink` espresso near-black, ink LIGHTENS as it gets less important (opposite of the dark theme). Semantic colors are **deepened** so they read on paper.
- **Ink is the primary action color**, not the accent: `.btn-accent` = ink bg + paper text (a black pill on cream). The per-workspace **accent** survives only as the deepened monogram tile (identity marker) + small semantic touches; chrome (tabs, FAB) is ink for guaranteed contrast across all 8 hues.
- **Printed-ledger structure.** Editorial mastheads (Bodoni), tracked-out overline section markers, hairline `.rule` rows, generous margins. Heroes are text-first magazine layouts, NOT boxes. Faint paper-tooth grain (`.grain`, ~0.06 multiply).

Anti-references: generic iOS-Settings lists; stacks of identical rounded cards; system-font UI; dark "fintech app" look; Bootstrap/Material shadows; AI-admin-panel. No many-slice pies, no decorative sparklines.

**Gotcha:** set `--accent` as a literal on the same element as `--ws-accent` (workspace wrapper + unauth pages), never `--accent: var(--ws-accent)` at `:root` — it resolves against an unset var.

## Design Principles

1. **Money never jitters.** Tabular figures everywhere; animate value _changes_ with a controlled roll, never reflow. Correctness of the number is sacred — motion serves legibility.
2. **It should feel native, not rendered.** View Transitions to morph list→detail, sheet presentations with spring physics, real material/vibrancy layering, safe-area aware. The wow is "wait, this is a website?"
3. **The approval screen is the centerpiece.** Answering a request should feel like replying to a message, not filling a form. Big photo, big amount, one gesture.
4. **One accent, rare and meaningful.** 60% surface / 30% secondary / 10% accent. Semantic color only where state must read instantly (green approved, red denied, amber pending, distinct sealed).
5. **Honest motion.** Fast (<200ms), exponential ease-out, no bounce. Respect `prefers-reduced-motion` with a beautiful static fallback. Every animation conveys a state change.
6. **Sealed is first-class for the owner, invisible to everyone else.** The owner always knows who can see what, in plain words.
